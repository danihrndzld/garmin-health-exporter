# Design Context — Garmin Data Exporter

Premise document for `frontend-design` and Impeccable skills. Keep every design
decision coherent with what's written here. Update when intent shifts; do not
edit to match accidental drift.

## Target Audience

Technically literate individuals exporting their own Garmin health data for
personal analysis, archival, or backup. They are comfortable with CSVs, file
paths, and command-line tools. They are *not* athletes looking for training
insights — this is not a sport app.

## Use Cases

1. **Scheduled batch export** — open the app weekly, enter credentials, click
   download, let it run. Primary path.
2. **Debug a failed login or export** — read the log, understand what broke,
   retry. The terminal panel exists for this.
3. **Inspect progress** — watch phases advance during a long run without
   guessing whether it's stuck.
4. **Open the output** — jump into Finder at the end without hunting for the
   folder.

## Brand Personality

Precise. Industrial. Professional instrumentation.

- Feels like a medical monitor, flight-test console, or modular synth readout.
- Reads as trustworthy under scrutiny — nothing decorative that implies the
  data is being massaged.
- Not gamified. Not sporty. No green-checkmark wellness copy. No badges.
- Confident but quiet — the UI does not celebrate itself.

## Aesthetic Direction

- Dark only. A warm, tinted near-black — not pure `#000`, not a blue-black.
- Sparse accents: one dominant red for primary/active state, a teal lineage
  for connected/success, an orange reserved for caution. Never all three
  competing at once.
- Type is a working instrument panel: a condensed display face for title
  slabs, a geometric UI face for controls, a mono for values and logs.
- Motion reinforces state, never decorates. Reduced-motion users must lose
  nothing functional.
- Surfaces feel etched, not glossed. No glassmorphism, no glow-for-glow's-sake.

## Visual Reference Anchors

- Hospital vital-signs monitors (ECG trace, steady dot pulse, muted chrome).
- Aerospace telemetry ground stations (dense but legible, monochrome + one
  alert color).
- Analog modular synth faceplates (labeled sections, mechanical rhythm,
  screen-printed typography).
- Early-2000s Bloomberg terminal density — minus the all-caps shouting.

## Anti-References

- Generic dark-mode SaaS dashboards (purple-to-blue gradients, glassy cards).
- Fitness apps (rings, confetti, badges, emoji).
- Developer-tool chic (neon cyan on pure black, Monaco everywhere).
- Anything that could plausibly be AI-generated from a "dark theme with red
  accents" prompt.

## Non-Negotiables

- WCAG 2.2 AA contrast on all text.
- `prefers-reduced-motion` fully respected.
- No new dependencies — plain CSS, inline SVG, vanilla JS.
- Layout is sidebar + terminal; this is not up for redesign.
- IPC surface is frozen; visuals may change, behavior may not.
